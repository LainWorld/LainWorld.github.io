Q：为什么软件开发周期总是预估的2-3倍

A：因为软件开发过程中总是会遇到各种突发情况，各种问题会随着进度的推进而不断冒出。

  1. 软件开发的思维模式不能像数学一样简单的计算，软件开发过程没有公式可以套，不是按照一定的步骤就能得到想要的结果。
  2. 开发环境与过程会极大的影响进度。
  3. 需求理解有差异，用户表达的和程序员理解的不一样，出现偏差。
  4. 项目负责人任务分配的时间不合理。 有的任务分配时间过长，浪费了时间，有的任务分配时间过短，不够时间去完成。
  5. 预估开发周期的时候没有考虑到一些特殊情况，比如遇到法定节假日放假等。
  6. 也有可能团队中出现短板或者团队配合度不够

Q2：：什么是分而治之

A： 把一个复杂的问题分成多个相同或相似的子问题，再把子问题分成更小的子问题直到最后子问题可以简单地直接求解，原问题的解即子问题的解的合并。把复杂的问题简单化。

分治策略：对于一个规模为n的问题，若该问题可以容易的解决(比如规模n较小)则直接解决，否则将其分解为k个规模较小的子问题，这些子问题互相独立且与原问题形式相同，递归地解决这些子问题，然后将各个子问题的解合并得到原问题的解。

如果原问题可以分割成k个子问题，1<k<=n，且这些子问题均可解并且利用这些子问题的解求出原问题的解，那么分治方法就是可行的。由分治法产生的子问题往往是原问题的较小模式，这就为使用递归技术提供了方便。反复应用分治手段，可以使子问题与原问题类型一致而其规模却不断缩小，最终使子问题缩小到很容易直接求出其解。这自然导致递归过程的产生。分治与递归经常同时应用在算法设计之中。

使用场景

  1. 计算机科学中的并行计算： 在计算机科学领域，分而治之被广泛应用于并行计算中。例如，在图像处理中，一个大型的图像可以被分解成多个小块，并在每个小块上进行并行处理，最后将结果组合起来以获得整个图像的结果。这种方法不仅可以加速图像处理的速度，还能减少处理过程中的错误率。

  2. 生产制造中的生产流程管理： 分而治之也被广泛应用于生产制造中的生产流程管理。例如，在汽车制造中，整个生产流程可以被分解成多个部分，每个部分负责不同的任务，例如焊接、喷涂、组装等。这样一来，工人们可以更好地专注于自己的任务，从而提高了生产效率和产品质量。

  3. 自然资源管理中的环保: 在自然资源管理中，分而治之 also 被广泛应用于环保方面。例如，在水资源管理中，一个大型的水文系统可以被分解成多个小区域，每个小区域可以独立进行监测和管理。这种方法不仅可以提高监测和管理的精度，还能有效地保护水资源，减轻环境压力。

Q3：了解WBS

A：

WBS是 Work Breakdown Structure的缩写，意为 工作分解结构。它是一种项目管理工具，用于将一个大型项目分解成一个个更小、更易管理的任务和子任务。WBS可以帮助团队将整个项目拆分为可控制的部分，从而更容易实现项目目标。

WBS通常采用层级结构来组织任务和子任务，每个层级表示不同的任务级别。最高层级通常代表整个项目，随后的层级则逐渐展开并细化各个任务和子任务。WBS图表通常以树形结构呈现，使得整个项目在视觉上更加清晰明了。

优点

  1. 明确项目范围：WBS能够帮助团队明确项目的范围，将整个复杂项目拆分为可管理的任务和子任务，从而更好地把握项目进度和完成情况。

  2. 便于任务分配：WBS通过将项目任务和子任务分解到不同的层级中，使得任务分配更加清晰明了，每个团队成员都知道自己需要完成的任务和目标。

  3. 提高效率：WBS能够帮助团队更好地规划项目计划，提高团队成员之间的协作效率，并有效地避免浪费时间和资源。

  4. 易于监督：WBS能够为项目经理提供清晰的项目进度和完成情况的概览，从而使得项目管理更加容易监督和跟踪。

与分而治之的区别

与“分而治之”类似，WBS也是一种将大型任务分解成小型任务以便于处理的方法。但是，WBS与“分而治之”的区别在于其使用范围和目的不同。具体来说：

  1. WBS通常用于项目管理中，而“分而治之”可以应用于各种类型的问题。

  2. WBS是一种专业的项目管理工具，它需要经验和技能才能正确实施和使用。而“分而治之”是一种普遍适用的思维方式，几乎任何人都可以使用。

  3. WBS强调对工作进行结构化分解，以便更好地控制项目进展，避免漏洞。而“分而治之”则更注重在一定程度上将问题分解并分配给不同的人或小组，以便快速高效地解决它们。

Q：软件工程是不是教会不怎么会写程序的人开发软件？你的观点？

A： 我认为软件工程并不是教会不怎么会写程序的人开发软件的课程。

软件工程的目标是通过系统的学习，让我们了解软件开发的整个过程，包括问题定义、可行性研究、需求分析、总体设计、详细设计、编码和单元测试、综合测试及运行维护等阶段。软件工程更倾向于教会我们如何用规范的管理方法去管理自己的开发过程，而不是仅仅教授如何编写软件代码。

编程能力有所欠缺的人可以通过专门性的编程语言课程来提升，而不是从另一学科中去学习。因此，我认为软件工程并不是一门教会不怎么会写程序的人开发软件的课程

Q：什么是DevOps？阅读以下材料，做好笔记•

https://www.zhihu.com/question/58702398

A：DevOps 是一种软件开发和运维的实践，旨在促进开发和运维之间的协作和整合。它结合了开发（Development）和运维（Operations）两个领域，以实现更快、更频繁、更可靠的软件交付。

DevOps 通过自动化软件交付流程和敏捷开发方法，实现了开发、测试和部署的紧密协作。开发人员可以通过持续集成和持续交付（CI/CD）工具，快速构建、测试和部署代码。运维人员可以利用监控、自动化、容器化等技术，管理和优化服务器、网络、存储和安全性。

DevOps 的核心是透明的、可扩展的软件开发和运维团队合作。通过这种合作，开发团队可以更早地识别和解决问题，而运维团队可以更加高效地支持软件交付。这种合作模式可以大大提高软件开发和运维的效率和质量，从而提高整个组织的业务绩效。

Q2：查阅相关资料，了解什么是scrum中的3355？

A：Scrum的3355是指Scrum团队用来检查和改进产品、过程和服务的核心步骤。它包括以下三个方面：

  1. 3个工件：Product Backlog、Sprint Backlog和 Sprint Self-Deployment（可交付物）。产品背景文件（Product Backlog）是团队正在开发的产品的详细列表；Sprint背景文件（Sprint Backlog）是团队计划在Sprint期间完成的产品功能列表；Sprint自我部署（Sprint Self-Deployment）是指将在Sprint期间完成的可交付物从Sprint Backlog转移到产品发布过程中的一系列步骤。
  2. 3个角色：Product Owner、Scrum Master和开发团队。产品负责人（Product Owner）定义产品的需求和优先级，Scrum Master负责管理和监控整个开发过程，开发团队则负责实现这些需求和优先级。
  3. 5个会议：Sprint Planning会议、Daily Scrum会议、Sprint Review会议、Retrospective会议和 Daily Standup会议。Sprint Planning会议是在Sprint开始时举行的，主要是计划和决定Sprint的目标和计划；Daily Scrum会议是每日举行的站会，是团队沟通和交流的重要场所，所有成员都要参加，包括产品负责人、Scrum Master和开发团队；Sprint Review会议是在每个Sprint结束时举行的，主要是评估已完成的产品功能和可交付物，并将结果汇报给产品负责人和Scrum Master；Retrospective会议是在项目结束时举行的，主要是回顾整个项目的进展和结果，并对未来的项目提出建议和计划；Daily Standup会议是每日举行的站会，是团队成员进行自我管理和准备的重要场所。

Scrum框架的核心思想是透明度、检查和改进。通过这三个核心步骤，Scrum团队能够更好地管理和监控整个开发过程，并及时发现和解决问题，以确保产品能够按时交付。

Q3：理解结对编程

A：

结对编程是一种编程教学方法，旨在通过两个程序员一起编程，相互学习、相互研究，从而提高代码质量和开发速度。这种方法可以促进团队成员之间的沟通和协作，帮助团队成员更好地了解彼此的工作方式和风格，从而提高团队合作能力和个人学习效率。

结对编程的好处有很多。从项目角度而言，结对编程可以提高代码质量和开发速度，因为两个程序员可以相互检查代码，发现潜在的问题或错误，并及时进行修复，从而提高代码质量。结对编程也可以促进团队成员之间的沟通和协作，因为两个程序员可以一起完成任务，比一个人单独工作更快。此外，结对编程还可以提高个人学习效率，因为两个程序员可以相互学习，相互研究，从而更快地学习和适应新技术。

在结对编程的过程中，也会遇到一些困难。例如，如何将结对编程有效地融合到开发流程中，以及如何有效地管理团队成员之间的沟通和协作。但是，总的来说，结对编程是一种非常有益的编程教学方法，可以显著提高代码质量和开发速度，促进团队合作，提高个人学习效率。

Q：软件及其相关的技术发展改变了哪些行业与领域，结合自身的体会和理解举例说明

A：基本是所有领域，但我觉得以下几个领域是表现最明显的

  1. 电子商务：随着互联网的普及和软件技术的进步，电子商务已经成为了一个蓬勃发展的行业。软件技术的发展使得电商平台能够提供更加便捷、安全和智能的购物体验，例如智能推荐、虚拟试衣等。
  2. 教育：随着在线教育的兴起，软件技术也改变了传统的教育模式。通过软件技术，学生可以方便地通过视频、语音、课堂讨论等方式接受课程。在线教育也有助于教师与学生之间的互动，更好地帮助学生理解课程内容。
  3. 金融：软件技术对金融业的影响也非常大。现在银行、保险公司和证券公司都使用软件来处理客户的账户和交易信息。这些软件可以提高效率，减少人工错误，并确保客户的资产安全。
  4. 医疗保健：随着健康意识的提高和大数据技术的应用，医疗保健领域也逐渐使用软件技术来提高效率和准确性。例如，医生可以使用软件来记录患者的病史和诊断结果，从而更好地帮助医生诊断和治疗疾病。
  5. 娱乐：随着游戏、视频和音乐等娱乐内容的数字化，软件技术也为娱乐产业带来了革命性的变化。例如，游戏玩家可以通过游戏手柄或者虚拟现实技术体验到更加真实的游戏场景，音乐爱好者可以使用软件来创作自己的音乐作品并分享给更多人。

软件工程师的基本素养

  1. 社会责任方面

    • 国家需求和社会需要

    • 职业道德和工匠精神

    • 可靠可信与自主可控

    • 伦理道德与社会影响

      其中更具体的可以表现为：

      • 只与声誉好的公司和组织建立联系。

      • 保证客户、雇主和主管知道在本道德规范中软件工程师所应承担的义务和责任。

      • 支持按照本规范要求去做的那些人。

      • 帮助发展一种有利于道德行为的组织环境。

      • 对任何有理由相信违反了本规范的事情均应向相应授权(者)机构报告。

      • 对他们工作的软件和相关文档应履行检测、纠正和报告错误等职责。

      • 只接受与职业资格或经验相称的酬报。

      • 准确地陈述他们工作的软件的特性，不但要避免错误的断言，而且要避免有理由被认为是欺骗的、误导的或令人怀疑的断言。

      • 不要以职业上的代价来发展自己的兴趣。

      • 服从所有管理他们工作的法规，使他们的工作与公众的健康、安全和福利要求相一致。

      • 以对民众事务具有建设性的服务来履行对社会的职业责任。

      • 促进公众对软件工程的了解。

        在专业方面：

        1. 扎实的计算机专业知识：这是软件工程师能够从事软件开发和实现最基本的前提，必须精通高等数学、离散数学、电子学、编程语言、数据结构等课程。
        2. 良好的语言表达能力和沟通能力：软件工程师经常需要和用户交流和沟通，必须具备清晰准确的表达能力和沟通能力，理解用户的需求和要求。
        3. 较强的工程经济分析能力：软件开发涉及到许多经济因素，例如成本、进度和质量等，软件工程师必须具备较强的工程经济分析能力，能够进行成本效益分析和投资效益预测。
        4. 团队精神和协作能力：软件开发通常是一个团队协作的过程，软件工程师必须具备团队精神，善于和其他成员合作，理解和尊重其他成员的观点和想法。
        5. 规范化，标准化的代码编写习惯：软件工程师必须具备良好的代码编写习惯，例如使用有意义的变量名，有效的注释和文档化代码等，这有助于提高代码的可读性和可维护性。
        6. 持续学习和创新能力：软件技术发展迅速，软件工程师必须具备持续学习和创新能力，跟踪最新的技术和趋势，并将其应用到自己的项目中，提高项目的竞争力。
        7. 责任心和安全意识：软件工程师必须具备高度的责任心，保证项目的质量和安全，了解安全风险和合规要求，采取相应的措施来确保项目的安全性。
